结论

  • 主命令面已覆盖：源码事实、路径浏览、索引生命周期、parser fallback、关系候选、hook/watch/serve 状态命令、SCIP 兼容导入、shell completions。
  • 在 RuoYi 这类多模块 Java 仓库上，基础 L0 查询与索引查询路径都可用；literal find 能切到 fresh text index 并显示 prefilter=trigram。
  • JS parser path 可工作：在 ruoyi-admin/src/main/resources/static/ruoyi/login.js 上，symbols、defs、calls、callers 都返回了预期结果。
  • 通过一个临时最小 SCIP JSON fixture，index import-scip 以及 precise symbols/defs/refs 也已验证通过。

重点发现

  • Warning Java 当时不在 tree-sitter parser 支持列表内。对 GenController.java 执行 defs GenController 时返回空结果且无 warning 文案，行为稳定但可读性不够强。
    2026-05-31 Resolved 已接入 tree-sitter-java；Java class/method definitions 和 method invocation callers 已支持。
  • Warning index build 会在目标仓库创建未跟踪的 .code-search/；随后 changed 会把它报出来，影响“工作区是否有真实源码变更”的语义。
  • Warning import-scip 目前绑定到当前 snapshot key。若 .code-search/ 让仓库从 commit:* 切到 worktree:*，precise 查询会错过刚导入的 SCIP 索引。本次用临时本地 ignore 验证了这是 snapshot 切换问题，而不是 SCIP 数据本身有误；验证后已撤销该本地 ignore。
